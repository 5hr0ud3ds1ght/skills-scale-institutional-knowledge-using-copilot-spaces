# OctoAcme Project Management Docs

This README provides a summary of OctoAcme's project management approach and links to the complete set of process documents.

## Overview

OctoAcme uses a structured, repeatable project management framework that applies to all cross-functional projects delivering product features, services, or integrations.

### Principles

- **Customer-first:** Prioritize customer value and usability.
- **Iterative delivery:** Deliver small, testable increments.
- **Clear ownership:** Each project has a named Project Manager (PM) and Product Lead.
- **Data-informed decisions:** Measure impact and iterate based on evidence.
- **Psychological safety:** Encourage feedback and learning.

### Lifecycle

1. **Initiation:** Define the problem statement, identify stakeholders, and establish a high-level timeline.
2. **Planning:** Define scope, resources, milestones, and dependencies.
3. **Execution:** Build, test, review, and iterate.
4. **Release:** Deploy, verify, and announce.
5. **Close & Retrospective:** Capture learnings and next steps.

### Key Artifacts

- Project Charter / One-pager
- Roadmap and Release Plan
- Sprint/Iteration Backlog
- Acceptance Criteria & Definition of Done
- Risk Register
- Retrospective notes and action items
